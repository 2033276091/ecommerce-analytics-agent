# 002 — 指标查询 — 近30天下单GMV与成交GMV对比

**问题：** 近30天的下单gmv和成交gmv分别是多少？

**意图类型：** 指标查询

**分析时间：** 2026-07-05

---

## 1. Schema 召回

| 类型 | 内容 |
|---|---|
| 表 | `fact_orders`（订单事实表） |
| 字段 | `gross_amount`（原始金额）、`paid_amount`（实付金额）、`order_status`（订单状态）、`order_date`（下单日期） |
| 匹配指标 | `gmv` — 标准口径为 `SUM(paid_amount) WHERE order_status = 'paid'`，即成交 GMV |
| 自定义指标 | 下单 GMV — 知识库无预定义口径，按业务惯例定义为 `SUM(gross_amount)`，不做状态过滤 |

---

## 2. SQL

**下单 GMV：**
```sql
SELECT ROUND(SUM(gross_amount), 2) AS order_gmv
FROM fact_orders
WHERE date(order_date) >= date('now', '-30 days');
```

**成交 GMV：**
```sql
SELECT ROUND(SUM(paid_amount), 2) AS paid_gmv
FROM fact_orders
WHERE order_status = 'paid'
  AND date(order_date) >= date('now', '-30 days');
```

---

## 3. SQL 校验

两条 SQL 均为只读 SELECT，无危险操作，校验通过 ✅

---

## 4. 查询结果

| 指标 | 金额（元） | 口径 |
|---|---|---|
| **下单 GMV** | **1,139,074.10** | 近 30 天全部 1,950 笔订单的原始金额 `SUM(gross_amount)` |
| **成交 GMV** | **943,193.19** | 近 30 天已支付 1,793 笔订单的实付金额 `SUM(paid_amount) WHERE order_status = 'paid'` |
| **差额** | **195,880.91** | 下单与成交之间的差距 |

---
