---

## 1. 意图识别

**意图类型**：报表生成

用户请求按「省份 + 性别 + 年龄段」三个维度生成包含实付金额和退款金额的交叉报表。

---

## 2. Schema 召回

### 使用到的表

| 表名 | 中文名 | 用途 |
|------|--------|------|
| dim_users | 用户维表 | 提供省份、性别、年龄段三个维度 |
| fact_orders | 订单事实表 | 提供实付金额（paid_amount） |
| fact_refunds | 退款事实表 | 提供退款金额（refund_amount） |

### 使用到的字段

| 表.字段 | 用途 |
|----------|------|
| dim_users.province | 维度：用户所在省份 |
| dim_users.gender | 维度：性别 |
| dim_users.age_group | 维度：年龄段 |
| fact_orders.paid_amount | 度量：实付金额 |
| fact_refunds.refund_amount | 度量：退款金额 |

### 匹配的指标口径

| 指标 | 口径 | 来源 |
|------|------|------|
| **实付金额** | `SUM(fact_orders.paid_amount)`，仅统计 `order_status = 'paid'` | GMV 口径 |
| **退款金额** | `SUM(fact_refunds.refund_amount)`，仅统计 `refund_status = 'approved'` | 退款金额口径 |

---

## 3. 生成的 SQL

```sql
SELECT 
    u.province,
    u.gender,
    u.age_group,
    ROUND(COALESCE(SUM(o.paid_amount), 0), 2) AS paid_amount,
    ROUND(COALESCE(SUM(r.refund_amount), 0), 2) AS refund_amount
FROM dim_users u
LEFT JOIN (
    SELECT user_id, SUM(paid_amount) AS paid_amount
    FROM fact_orders
    WHERE order_status = 'paid' 
      AND date(order_date) >= date('now', '-7 days')
    GROUP BY user_id
) o ON u.user_id = o.user_id
LEFT JOIN (
    SELECT user_id, SUM(refund_amount) AS refund_amount
    FROM fact_refunds
    WHERE refund_status = 'approved' 
      AND date(refund_date) >= date('now', '-7 days')
    GROUP BY user_id
) r ON u.user_id = r.user_id
GROUP BY u.province, u.gender, u.age_group
ORDER BY u.province, u.gender, u.age_group
```

### SQL 设计说明

- 两个度量值来自不同表、不同时间字段：`fact_orders.order_date`（下单时间）和 `fact_refunds.refund_date`（退款时间），因此先用子查询按 `user_id` 分别聚合，再通过 `dim_users` 关联维度。
- 使用 `LEFT JOIN` 保证即使用户在该时间段无支付或无退款也能出现在报表中（金额显示为 0）。
- 严格遵循口径：实付金额仅统计 `order_status = 'paid'`，退款金额仅统计 `refund_status = 'approved'`。

4. **退款情况**：整体金额退款率约 9.47%，退款集中在部分高消费用户群，其中山东男性 25-34 岁段退款金额最高（¥2,738.99）。

## 7. 下一步分析建议

1. **下钻分析**：点击某个省份-性别-年龄段组合，查看具体品类的销售/退款构成。
2. **趋势分析**：将时间范围扩展至近 30 天，观察各维度实付金额和退款金额的日趋势变化。
3. **退款归因**：针对退款金额较高的组合（如山东男 25-34），分析退款原因分布，判断是否存在商品质量或尺码问题。
4. **转化漏斗**：建议结合流量表 `fact_traffic_daily`，按省份分析「访客 → 支付」转化率，评估各区域流量质量。
5. **客单价对比**：增加客单价指标，对比不同维度的客单价差异，挖掘高价值人群特征。
