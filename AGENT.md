# 电商智能问数 Agent 项目计划

## 项目背景

这是一个面向电商经营分析的智能问数 Agent，能理解业务问题、生成 SQL、查询数据、做指标解释、发现异常，并能展示它如何降低运营/数据分析成本。

围绕常见电商经营问题，完成业务场景、模拟数据、表结构、指标口径、Text-to-SQL、SQL 校验、查询执行和业务解释。

## 项目定位

面向电商运营、商品运营、数据分析师的智能问数 Agent。用户用自然语言提问，系统理解问题、选择相关数据表和指标口径、生成安全 SQL、查询本地模拟数据库，并用业务语言解释结果。

这个项目不是泛泛的聊天机器人，而是一个小型数据资产问答系统：

- 有电商业务主题域。
- 有可解释的指标口径。
- 有 Schema 和字段说明。
- 有 SQL 生成与安全校验。
- 有查询结果和业务解释。
- 有可用于面试讲解的架构、数据链路和案例。

## MVP 范围

### 第一阶段：业务场景、模拟数据、表结构

目标：搭建一个可信的电商经营数据底座。

常用数据域：

- 用户域：用户、注册时间、地区、会员等级。
- 商品域：商品、品类、品牌、成本价、销售价。
- 交易域：订单、订单明细、支付金额、优惠金额、订单状态。
- 流量域：访问量、访客数、加购人数、下单人数、支付人数。
- 售后域：退款单、退款金额、退款原因、退款状态。
- 渠道域：自然流量、搜索、推荐、直播、广告等来源。

核心指标：

- GMV：已支付订单的商品成交金额。
- 支付订单数：已支付订单数量。
- 支付用户数：有支付行为的用户数。
- 客单价：GMV / 支付订单数。
- 件单价：GMV / 销售件数。
- 转化率：支付用户数 / 访客数。
- 加购率：加购人数 / 访客数。
- 退款率：退款金额 / GMV，或退款订单数 / 支付订单数。
- 复购率：一段时间内购买 2 次及以上用户数 / 支付用户数。

### 第二阶段：智能问数链路

目标：让系统能回答常见经营问题。

流程：

1. 问题理解：判断用户是在查指标、做对比、看趋势、找原因还是生成简报。
2. Schema 召回：根据问题匹配相关表、字段、指标口径。
3. SQL 生成：只生成 SELECT 查询。
4. SQL 安全校验：禁止 DELETE、UPDATE、INSERT、DROP、ALTER 等危险语句。
5. 查询执行：在本地数据库执行 SQL。
6. 结果解释：用电商业务语言解释查询结果。
7. 追问建议：给出下一步可分析方向。

### 第三阶段：面试展示材料

目标：把项目讲得像真实业务落地。

交付物：

- README：项目背景、架构、运行方式、演示问题。
- 数据模型文档：表结构、字段解释、指标口径。
- 示例问题集：覆盖 GMV、转化率、退款率、复购、品类排行、区域分析。
- 演示脚本：周一向面试官展示时的讲解顺序。

## 推荐演示问题

- 昨天 GMV 是多少，环比前一天变化多少？
- 最近 7 天哪个品类 GMV 最高？
- 华东地区女装类目的退款率是否偏高？
- 最近 30 天复购率最高的用户群体是什么？
- 搜索渠道和直播渠道的转化率分别是多少？
- 本周 GMV 下降主要是流量、转化率还是客单价导致？
- 给我生成一份本周经营简报。

## 周末执行顺序

1. 设计业务场景、表结构、指标口径。
2. 生成模拟数据和本地数据库。
3. 实现 SQL 查询执行工具。
4. 实现指标与 Schema 知识库。
5. 实现自然语言问题到 SQL 的 Agent 链路。
6. 加 SQL 安全校验和错误修复。
7. 整理 README、演示问题和面试讲稿。
